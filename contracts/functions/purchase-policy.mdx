The `purchasePolicy` function allows farmers to purchase drought insurance by paying a premium in ETH and registering their farm location.

## Function Signature

```solidity
function purchasePolicy(int256 lat, int256 long) external payable
```

## Parameters

<ParamField path="lat" type="int256" required>
  Latitude coordinate of the insured farm location. This should be a scaled integer representation (e.g., multiply by 1e6 for precision).
</ParamField>

<ParamField path="long" type="int256" required>
  Longitude coordinate of the insured farm location. This should be a scaled integer representation (e.g., multiply by 1e6 for precision).
</ParamField>

<ParamField path="msg.value" type="uint256" required>
  The premium amount in wei sent with the transaction. Must be greater than 0.
</ParamField>

## Implementation

```solidity
function purchasePolicy(int256 lat, int256 long) external payable {
    // Assuming ETH native is used for simplicity, or we can use USDC.
    // For the hackathon, we use native base ETH for simplicity of the contract,
    // pretending it's a USD peg or using OnchainKit for seamless handling.
    require(msg.value > 0, "Premium must be greater than 0");
    
    bytes32 policyId = keccak256(abi.encodePacked(msg.sender, lat, long, block.timestamp));
    
    policies[policyId] = Policy({
        farmer: msg.sender,
        lat: lat,
        long: long,
        premiumPaid: msg.value,
        isActive: true
    });
    
    activePolicyIds.push(policyId);
    
    emit PolicyPurchased(policyId, msg.sender, lat, long, msg.value);
}
```

## Return Values

This function does not return a value but emits a `PolicyPurchased` event.

<ResponseField name="PolicyPurchased" type="event">
  Event containing the policy ID, farmer address, location coordinates, and premium amount
</ResponseField>

## Event Emitted

```solidity
event PolicyPurchased(
    bytes32 indexed policyId,
    address indexed farmer,
    int256 lat,
    int256 long,
    uint256 premium
);
```

<ResponseField name="policyId" type="bytes32" required>
  Unique identifier for the newly purchased policy, generated by hashing the farmer address, coordinates, and timestamp
</ResponseField>

<ResponseField name="farmer" type="address" required>
  Address of the farmer who purchased the policy (msg.sender)
</ResponseField>

<ResponseField name="lat" type="int256" required>
  Latitude coordinate stored in the policy
</ResponseField>

<ResponseField name="long" type="int256" required>
  Longitude coordinate stored in the policy
</ResponseField>

<ResponseField name="premium" type="uint256" required>
  Premium amount paid in wei
</ResponseField>

## Usage Example

Using ethers.js:

```javascript
import { ethers } from 'ethers';

// Connect to contract
const contract = new ethers.Contract(contractAddress, abi, signer);

// Purchase policy with 0.01 ETH premium
const lat = 34_520000; // 34.52° scaled by 1e6
const long = -117_185000; // -117.185° scaled by 1e6
const premium = ethers.parseEther('0.01');

const tx = await contract.purchasePolicy(lat, long, {
  value: premium
});

const receipt = await tx.wait();

// Extract policy ID from event
const event = receipt.logs.find(log => log.eventName === 'PolicyPurchased');
const policyId = event.args.policyId;

console.log('Policy purchased:', policyId);
```

## Requirements

<ResponseField name="Premium > 0" type="requirement">
  The transaction must include ETH (msg.value > 0), otherwise it reverts with "Premium must be greater than 0"
</ResponseField>

<ResponseField name="Valid Coordinates" type="requirement">
  Latitude and longitude should be valid geographic coordinates, scaled appropriately
</ResponseField>

## Policy ID Generation

The policy ID is deterministically generated:

```solidity
bytes32 policyId = keccak256(abi.encodePacked(
    msg.sender,
    lat,
    long,
    block.timestamp
));
```

This ensures:
- **Uniqueness**: Including `block.timestamp` prevents duplicate IDs
- **Deterministic**: Given the inputs, the ID can be predicted
- **Collision-resistant**: Keccak256 provides strong hash properties

<Note>
  The policy ID is returned in the `PolicyPurchased` event. Frontend applications should listen for this event to track the policy ID.
</Note>

## State Changes

1. Creates a new `Policy` struct in the `policies` mapping
2. Adds the policy ID to the `activePolicyIds` array
3. Transfers the premium (msg.value) to the contract balance

## Gas Considerations

- **Estimated Gas**: ~100,000 - 150,000 gas
- **Variable Costs**: Gas cost increases slightly with the number of active policies due to array push

<Tip>
  Use `estimateGas` before submitting the transaction to get an accurate gas estimate.
</Tip>

## Security

- **No Access Control**: Anyone can purchase a policy
- **No Validation**: The contract does not validate coordinate ranges
- **No Duplicate Prevention**: The same farmer can purchase multiple policies for the same location

<Warning>
  In production, consider adding validation for coordinate ranges and potentially limiting policies per address.
</Warning>
