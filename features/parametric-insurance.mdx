## What is Parametric Insurance?

Parametric insurance is a type of coverage that pays out based on **predetermined trigger events** rather than actual losses. Instead of assessing damage after an event, parametric policies automatically pay when objective, measurable conditions are met.

<CardGroup cols={2}>
  <Card title="Trigger-Based" icon="bolt">
    Payouts activate when specific conditions are met (e.g., rainfall < 10mm)
  </Card>
  <Card title="No Loss Assessment" icon="ban">
    No need to prove or document actual crop damage
  </Card>
  <Card title="Instant Settlement" icon="clock">
    Funds transfer automatically within seconds of trigger
  </Card>
  <Card title="Fully Transparent" icon="eye">
    All trigger conditions and data sources are public
  </Card>
</CardGroup>

### Payout Structure

When triggered, payouts are deterministic and immediate:

```solidity
// Fixed payout multiplier
function executePayout(bytes32 policyId) external onlyCRE {
    Policy storage policy = policies[policyId];
    require(policy.isActive, "Policy is not active");
    
    policy.isActive = false;
    
    // Payout is always 10x the premium
    uint256 payoutAmount = policy.premiumPaid * 10;
    
    require(address(this).balance >= payoutAmount, "Insufficient balance");
    
    (bool success, ) = policy.farmer.call{value: payoutAmount}("");
    require(success, "Payout transfer failed");
    
    emit PayoutExecuted(policyId, policy.farmer, payoutAmount);
}
```

**Payout Characteristics:**

- **Amount**: 10x the premium paid
- **Speed**: Executes in single transaction (2-5 seconds)
- **Finality**: Irreversible once confirmed on-chain
- **Transparency**: `PayoutExecuted` event logged publicly

## Data Aggregation & Accuracy

### Consensus Median Aggregation

AridGuard uses the CRE SDK's built-in aggregation to ensure data accuracy:

```typescript
import { consensusMedianAggregation } from "@chainlink/cre-sdk";

const fetchAndParse = (sendRequester: HTTPSendRequester, config: Config, apiKey: string) => {
  const url = `${config.weatherApiUrl}&appid=${apiKey}`;
  
  const response = sendRequester
    .sendRequest({ url: url, method: "GET" })
    .result();
  
  if (!ok(response)) {
    throw new Error(`HTTP request failed with status: ${response.statusCode}`);
  }
  
  const externalResp = json(response) as any;
  const rainfall = externalResp.rain ? externalResp.rain["1h"] || 0 : 0;
  return rainfall;
};

const httpCapability = new cre.capabilities.HTTPClient();
const medianRainfall = httpCapability
  .sendRequest(runtime, fetchAndParse, consensusMedianAggregation())
  (runtime.config, apiKey)
  .result();

runtime.log(`Aggregated median rainfall: ${medianRainfall} mm`);
```

**How Median Aggregation Works:**

<Steps>
  <Step title="Multiple Requests">
    CRE makes parallel requests to weather API from different nodes
  </Step>
  <Step title="Collect Responses">
    Each node returns its rainfall measurement
  </Step>
  <Step title="Sort Values">
    All responses sorted numerically: [5mm, 7mm, 8mm, 9mm, 15mm]
  </Step>
  <Step title="Select Median">
    Middle value selected: **8mm**
  </Step>
  <Step title="Outlier Rejection">
    Anomalies (15mm) automatically excluded
  </Step>
</Steps>

**Why Median > Average:**

```
Scenario: 5 measurements
Data points: 8mm, 8mm, 9mm, 9mm, 100mm (faulty sensor)

Average: (8+8+9+9+100)/5 = 26.8mm ❌ Incorrect
Median: [8, 8, 9, 9, 100] → 9mm ✅ Accurate
```

Median aggregation is **robust against outliers**, making it ideal for real-world data with potential sensor errors or API glitches.

### Data Parsing

OpenWeather API returns structured JSON:

```json
{
  "coord": { "lon": 36.0, "lat": 0.05 },
  "weather": [{ "id": 500, "main": "Rain", "description": "light rain" }],
  "main": { "temp": 22.5, "humidity": 65 },
  "rain": { "1h": 2.5 },
  "dt": 1632150000
}
```

AridGuard extracts the 1-hour rainfall measurement:

```typescript
const externalResp = json(response) as any;
const rainfall = externalResp.rain ? externalResp.rain["1h"] || 0 : 0;
return rainfall; // Returns 2.5 (mm)
```

If no rain is detected, the API omits the `rain` field entirely, so we default to `0`.

## Risk Management

### Basis Risk

The primary risk in parametric insurance is **basis risk**: mismatch between trigger and actual loss.

**Example Scenarios:**

1. **False Positive** (Payout without loss):
   - Trigger: Rainfall < 10mm detected
   - Reality: Farm has irrigation system, no damage occurred
   - Result: Farmer receives payout despite no loss

2. **False Negative** (Loss without payout):
   - Trigger: Rainfall 11mm (just above threshold)
   - Reality: Localized drought at farm, crops damaged
   - Result: Farmer suffers loss but receives no payout

**Mitigation Strategies:**

<Accordion title="Precise Trigger Calibration">
  Use historical data to set thresholds that maximize correlation between trigger events and actual losses:
  
  ```python
  # Analysis of historical data
  drought_years = [2011, 2015, 2017, 2022]
  avg_rainfall_during_drought = 8.5mm
  threshold = avg_rainfall_during_drought * 1.2  # 10.2mm buffer
  ```
</Accordion>

<Accordion title="Multi-Parameter Triggers">
  Combine multiple weather indicators:
  
  - Rainfall < 10mm **AND**
  - Temperature > 35°C **AND**
  - Humidity < 30%
  
  Reduces false positives significantly.
</Accordion>

<Accordion title="Satellite Verification">
  Use satellite imagery to confirm crop stress:
  
  - NDVI (Normalized Difference Vegetation Index) analysis
  - Soil moisture remote sensing
  - Thermal imaging for heat stress
</Accordion>

<Accordion title="Local Weather Stations">
  Deploy IoT weather sensors on farms:
  
  - More accurate than regional data
  - Real-time monitoring
  - Chainlink External Adapters for on-chain integration
</Accordion>

### Smart Contract Risk

**Risks:**
- Contract bugs leading to locked funds
- Reentrancy attacks on payout function
- Unauthorized access to `executePayout`

**Mitigations:**

```solidity
// Access control
modifier onlyCRE() {
    require(msg.sender == creAddress, "Only CRE can execute this");
    _;
}

// Reentrancy protection
function executePayout(bytes32 policyId) external onlyCRE {
    Policy storage policy = policies[policyId];
    require(policy.isActive, "Policy is not active");
    
    // State changes BEFORE external call
    policy.isActive = false;
    
    uint256 payoutAmount = policy.premiumPaid * 10;
    require(address(this).balance >= payoutAmount, "Insufficient balance");
    
    // External call (potential reentrancy)
    (bool success, ) = policy.farmer.call{value: payoutAmount}("");
    require(success, "Payout transfer failed");
    
    emit PayoutExecuted(policyId, policy.farmer, payoutAmount);
}
```

<Warning>
  Always audit smart contracts before production deployment. AridGuard is a hackathon demo and has not undergone formal security audits.
</Warning>
