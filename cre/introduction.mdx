## What is CRE?

The Chainlink Runtime Environment (CRE) is a decentralized oracle framework that enables secure, consensus-driven data aggregation and smart contract automation. AridGuard uses CRE to build a parametric insurance oracle that:

- Fetches real-time weather data from external APIs
- Aggregates rainfall measurements across multiple oracle nodes using consensus
- Automatically triggers insurance payouts when drought conditions are detected
- Executes blockchain transactions on Base Sepolia testnet

## How AridGuard Uses CRE

AridGuard's CRE workflow implements a fully automated parametric insurance system:

1. **Scheduled Monitoring**: A cron trigger runs at regular intervals to check weather conditions
2. **Data Fetching**: HTTPClient capability fetches rainfall data from OpenWeather API
3. **Consensus Aggregation**: Multiple oracle nodes fetch data and compute the median rainfall
4. **Threshold Detection**: The workflow checks if rainfall falls below the drought threshold
5. **Smart Contract Interaction**: When drought is detected, EVMClient reads active policies and executes payouts

## Workflow Architecture

```typescript
const initWorkflow = (config: Config) => {
  const cron = new cre.capabilities.CronCapability();
  return [cre.handler(cron.trigger({ schedule: config.schedule }), onCronTrigger)];
};
```

The workflow consists of:

- **Trigger**: CronCapability schedules automated checks
- **Handler**: `onCronTrigger` function processes each execution
- **Capabilities**: HTTPClient for API calls, EVMClient for blockchain interactions
- **Consensus**: Median aggregation across decentralized oracle nodes
