## Overview

The Weather API integration fetches real-time rainfall data using CRE's HTTPClient capability. AridGuard uses OpenWeather API by default, but supports other weather providers.

## OpenWeather API

AridGuard's default weather data provider.

### API Endpoint

```
https://api.openweathermap.org/data/2.5/weather?q={location}&units=metric&appid={api_key}
```

### Getting API Key

<Steps>
  <Step title="Create Account">
    Sign up at [openweathermap.org](https://openweathermap.org/)
  </Step>
  
  <Step title="Generate API Key">
    Navigate to API Keys section in your account dashboard and create a new key.
  </Step>
  
  <Step title="Verify Key">
    Test the key with a sample request:
    ```bash
    curl "https://api.openweathermap.org/data/2.5/weather?q=Nairobi&units=metric&appid=YOUR_KEY"
    ```
  </Step>
  
  <Step title="Configure in Workflow">
    Add to environment variables:
    ```bash
    export CRE_OPENWEATHER_API_KEY="your_api_key_here"
    ```
  </Step>
</Steps>

### API Response Format

OpenWeather returns JSON with rainfall data:

```json
{
  "coord": { "lon": 36.8167, "lat": -1.2833 },
  "weather": [
    {
      "id": 501,
      "main": "Rain",
      "description": "moderate rain",
      "icon": "10d"
    }
  ],
  "main": {
    "temp": 22.5,
    "feels_like": 22.8,
    "temp_min": 21.0,
    "temp_max": 24.0,
    "pressure": 1013,
    "humidity": 78
  },
  "rain": {
    "1h": 8.5
  },
  "name": "Nairobi"
}
```

### Extracting Rainfall Data

The workflow extracts rainfall from the `rain.1h` field:

```typescript
const fetchAndParse = (sendRequester: HTTPSendRequester, config: Config, apiKey: string) => {
  const url = config.weatherApiUrl.includes("?") 
    ? `${config.weatherApiUrl}&appid=${apiKey}` 
    : `${config.weatherApiUrl}?appid=${apiKey}`;

  const response = sendRequester
    .sendRequest({ url: url, method: "GET" })
    .result();

  if (!ok(response)) {
    throw new Error(`HTTP request failed with status: ${response.statusCode}`);
  }

  const externalResp = json(response) as any;
  const rainfall = externalResp.rain ? externalResp.rain["1h"] || 0 : 0;
  return rainfall;
};
```

### Handling Missing Data

When no rainfall is detected, the API may omit the `rain` field:

```typescript
const rainfall = externalResp.rain ? externalResp.rain["1h"] || 0 : 0;
```

This returns `0` mm if:
- The `rain` field is absent
- The `rain.1h` field is absent
- The `rain.1h` value is `null` or `undefined`

## HTTPClient Implementation

The CRE HTTPClient provides consensus-driven data fetching:

```typescript
const httpCapability = new cre.capabilities.HTTPClient();
const medianRainfall = httpCapability
  .sendRequest(runtime, fetchAndParse, consensusMedianAggregation())(runtime.config, apiKey)
  .result();

runtime.log(`Aggregated median rainfall: ${medianRainfall} mm`);
```

### Consensus Process

1. **Fetch**: Each oracle node calls OpenWeather API independently
2. **Parse**: Each node extracts the rainfall value
3. **Submit**: All nodes submit their readings to the consensus layer
4. **Aggregate**: CRE computes the median across all submissions
5. **Result**: The median value is used for threshold comparison

<Info>
  Median aggregation prevents individual node failures or API inconsistencies from affecting the result. Even if one node gets corrupted data, the median remains accurate.
</Info>

## Configuration

Weather API settings in `config.json`:

```json
{
  "weatherApiUrl": "https://api.openweathermap.org/data/2.5/weather?q=Nairobi&units=metric",
  "thresholdRainfall": 10
}
```

### Location Parameter

Specify city name:
```
?q=Nairobi
```

Or use coordinates:
```
?lat=-1.2833&lon=36.8167
```

### Units Parameter

<ParamField path="units" type="string" default="metric">
  - **metric**: Celsius, mm (recommended)
  - **imperial**: Fahrenheit, inches
  - **standard**: Kelvin (default if omitted)
</ParamField>

## Error Handling

### HTTP Errors

```typescript
if (!ok(response)) {
  throw new Error(`HTTP request failed with status: ${response.statusCode}`);
}
```

Common error codes:
- **401**: Invalid API key
- **404**: Location not found
- **429**: Rate limit exceeded
- **500**: API server error

### Network Failures

CRE automatically retries failed requests:

```typescript
try {
  const response = sendRequester
    .sendRequest({ url: url, method: "GET" })
    .result();
} catch (error) {
  runtime.log(`API request failed: ${error.message}`);
  throw error;
}
```

### Validation

Validate rainfall values before using:

```typescript
const rainfall = externalResp.rain ? externalResp.rain["1h"] || 0 : 0;

if (typeof rainfall !== 'number' || rainfall < 0) {
  throw new Error(`Invalid rainfall value: ${rainfall}`);
}

return rainfall;
```

### Tomorrow.io

Enterprise weather API:

```json
{
  "weatherApiUrl": "https://api.tomorrow.io/v4/weather/realtime?location=Nairobi"
}
```

Parsing example:

```typescript
const rainfall = externalResp.data?.values?.precipitationIntensity || 0;
return rainfall;
```

### Custom Weather API

Integrate any REST API:

```typescript
const fetchAndParse = (sendRequester: HTTPSendRequester, config: Config, apiKey: string) => {
  const response = sendRequester
    .sendRequest({ 
      url: config.weatherApiUrl,
      method: "GET",
      headers: {
        "Authorization": `Bearer ${apiKey}`,
        "Content-Type": "application/json"
      }
    })
    .result();

  if (!ok(response)) {
    throw new Error(`HTTP request failed with status: ${response.statusCode}`);
  }

  const externalResp = json(response) as any;
  
  // Custom parsing logic for your API
  const rainfall = externalResp.your_field_path || 0;
  return rainfall;
};
```

## Rate Limiting

### OpenWeather Limits

Free tier:
- **60 calls/minute**
- **1,000,000 calls/month**

With consensus of 5 nodes running hourly:
- **5 calls/hour = 120 calls/day = 3,600 calls/month**
- Well within free tier limits

### Handling Rate Limits

Implement exponential backoff:

```typescript
const fetchWithRetry = (sendRequester: HTTPSendRequester, url: string, maxRetries: number = 3) => {
  for (let i = 0; i < maxRetries; i++) {
    const response = sendRequester.sendRequest({ url, method: "GET" }).result();
    
    if (ok(response)) {
      return response;
    }
    
    if (response.statusCode === 429) {
      // Rate limited, wait before retry
      const waitTime = Math.pow(2, i) * 1000; // Exponential backoff
      runtime.log(`Rate limited, retrying in ${waitTime}ms...`);
      // Note: CRE SDK handles delays internally
    } else {
      throw new Error(`HTTP request failed with status: ${response.statusCode}`);
    }
  }
  
  throw new Error("Max retries reached");
};
```
