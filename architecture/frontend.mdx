## Key Implementation Details

### Hydration Mismatch Prevention

```typescript
const [mounted, setMounted] = useState(false);

useEffect(() => {
  setMounted(true);
}, []);

if (!mounted) {
  return <LoadingPlaceholder />;
}
```

**Why?** Next.js renders on server (no wallet state) then hydrates on client (with wallet state). Without this check, you get hydration mismatches.

### Transaction State Management

```typescript
// Track transaction submission
const { data: hash, isPending: isWritePending, writeContract } = useWriteContract();

// Track blockchain confirmation
const { isLoading: isConfirming, isSuccess: isConfirmed } = useWaitForTransactionReceipt({ hash });

// Combined loading state
const loading = isWritePending || isConfirming;
```

**States:**
1. **Idle**: No transaction
2. **Writing** (`isWritePending`): User confirming in wallet
3. **Confirming** (`isConfirming`): Waiting for block confirmation
4. **Confirmed** (`isConfirmed`): Transaction successful

### Dynamic UI Based on State

```typescript
{mounted ? (
  isConnected ? (
    purchased ? (
      <SuccessMessage />
    ) : (
      <PurchaseButton />
    )
  ) : (
    <ConnectWalletPrompt />
  )
) : (
  <LoadingPlaceholder />
)}
```

**Flow:**
- Not mounted → Loading placeholder
- Not connected → Connect wallet prompt
- Connected, not purchased → Purchase button
- Purchased → Success message
