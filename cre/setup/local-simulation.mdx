## Overview

Local simulation allows you to run and test your CRE workflow on your development machine without deploying to the Chainlink oracle network. This enables rapid iteration and debugging.

## Workflow Configuration

Ensure your `workflow.yaml` is configured for local simulation:

```yaml
local-simulation:
  user-workflow:
    workflow-name: "aridguard-local"
  workflow-artifacts:
    workflow-path: "./main.ts"
    config-path: "./config.json"
    secrets-path: "./secrets.yaml"
```

<Info>
  The `workflow-path`, `config-path`, and `secrets-path` are relative to the directory where you run the simulation command.
</Info>

## Running Local Simulation

### Basic Simulation

Run your workflow locally:

```bash
chainlink cre local simulate
```

This command:
1. Loads `workflow.yaml` configuration
2. Compiles `main.ts` TypeScript code
3. Reads `config.json` and `secrets.yaml`
4. Executes the workflow in a local runtime environment
5. Simulates oracle consensus (if applicable)

### Simulation Output

Expected console output:

```

[CRE] Compiling TypeScript...
[CRE] Initializing runtime...
[CRE] Executing cron trigger...
[WORKFLOW] Aggregated median rainfall: 8.5 mm

[WORKFLOW] Found active policy ID: 0x1234...5678. Executing payout transaction on Base...
[CRE] Workflow completed successfully
```

## Simulation Modes

### Single Execution

Run the workflow once:

```bash
chainlink cre local simulate --once
```

Useful for testing specific scenarios or debugging.

### Scheduled Execution

Run according to cron schedule:

```bash
chainlink cre local simulate --follow-schedule
```

This respects the `schedule` parameter in `config.json` and runs continuously.

### Mock Mode

Simulate without external API calls:

```bash
chainlink cre local simulate --mock-http
```

Returns mock data for HTTP requests, useful for testing without API limits.

## Debugging

### Enable Verbose Logging

Get detailed execution logs:

Add logging throughout your workflow:

const onCronTrigger = (runtime: Runtime<Config>) => {

  // ... rest of workflow

## Testing Scenarios

### Test Drought Condition

Modify `config.json` to force drought detection:

```json
{
  "schedule": "* * * * *",
  "weatherApiUrl": "https://api.openweathermap.org/data/2.5/weather?q=Nairobi&units=metric",
  "thresholdRainfall": 100
}
```

Setting a high threshold ensures rainfall will be below it.

### Test Normal Condition

Set a low threshold to simulate normal conditions:

```json
{
  "thresholdRainfall": 0
}
```

Rainfall should always exceed this threshold.

### Test Different Locations

Change the weather API location:

```json
{
  "weatherApiUrl": "https://api.openweathermap.org/data/2.5/weather?q=London&units=metric"
}
```

Test with various geographic regions.

## Mock HTTP Responses

For testing without API calls, create mock responses:

```typescript
const fetchAndParse = (sendRequester: HTTPSendRequester, config: Config, apiKey: string) => {
  // Development mock
  if (process.env.NODE_ENV === 'development') {
    return 5.0; // Mock rainfall value
  }
  
  // Production code
  const url = config.weatherApiUrl.includes("?") 
    ? `${config.weatherApiUrl}&appid=${apiKey}` 
    : `${config.weatherApiUrl}?appid=${apiKey}`;
    
  const response = sendRequester
    .sendRequest({ url: url, method: "GET" })
    .result();
    
  // ... parse response
};
```

## Consensus Simulation

Local simulation can simulate multi-node consensus:

```bash
chainlink cre local simulate --nodes 5
```

This simulates 5 oracle nodes fetching data independently and computing consensus.

### Median Aggregation Test

With `--nodes 5`, the simulator:
1. Executes `fetchAndParse` 5 times with slight variations
2. Collects all 5 rainfall readings
3. Computes the median value
4. Uses median for threshold comparison

Example output:
```
[Node 1] Rainfall: 8.2 mm
[Node 2] Rainfall: 8.7 mm
[Node 3] Rainfall: 8.5 mm
[Node 4] Rainfall: 8.3 mm
[Node 5] Rainfall: 8.9 mm
[Consensus] Median rainfall: 8.5 mm
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Simulation fails to start">
    Check that all required files exist:
    ```bash
    ls -la main.ts config.json secrets.yaml workflow.yaml
    ```
    
    Verify TypeScript compiles:
    ```bash
    npx tsc --noEmit
    ```
  </Accordion>
  
  <Accordion title="Secret not found error">
    Ensure environment variable is set:
    ```bash
    echo $CRE_OPENWEATHER_API_KEY
    ```
    
    Set it if missing:
    ```bash
    export CRE_OPENWEATHER_API_KEY="your_api_key"
    ```
  </Accordion>
  
  <Accordion title="HTTP request fails">
    Test API endpoint manually:
    ```bash
    curl "https://api.openweathermap.org/data/2.5/weather?q=Nairobi&units=metric&appid=YOUR_KEY"
    ```
    
    Check for:
    - Invalid API key
    - Rate limiting
    - Network connectivity
    - Malformed URL
  </Accordion>
  
  <Accordion title="Contract call fails">
    Verify contract address and network:
    ```typescript
    runtime.log(`Contract: ${contractAddress}`);
    runtime.log(`Network: ${chainName}`);
    ```
    
    Ensure:
    - Contract is deployed to the network
    - ABI matches deployed contract
    - Network is reachable
  </Accordion>
</AccordionGroup>
